Linux Kernel
provides a level of abstraction between the device hardware and contains essential hardware drivers like camera , keyboard , display , and networking.
Libraries
- open sources browser engine Webkit. LIBC , SQLite Database

Android Library
- android.app - provides access to application model and is cornerstone of all Android App.
- android.content - Facilitates content access , publishing and messaging between Apps and App components
- android.database - access data published by content providers , includes SQLite database management classes
- android.opengl - Java interface to OpenGl ES 3D graphics rendering API
- android.os - access to standard operating system services includes messages , system services , and interprocess communication.
- android.text - render and manipulate text on device display.
- android.view - fundamental building blocks of application user interfaces
- android.widget - pre-built interface components such as button , labels , list views , layout managers , radio button, etc..
- android.webkit - set of classes to allow web browsing capabilities to built into applications.
Android Runtime
- Dalvik Virtual Machine - JVM for android
- Dalvik VM make use of Linux core features
like memory management and multi-threading
- Dalvik VM enables every Android Apps to run its own process. with its own instance of Dalvik virtual machine.
Application Framework
Apps framework provides higher-level services to applications in the form of Java classes.
- Activity Manager - Controls all aspects of application lifecycle and activity stack.
- Content Providers - Allows application to publish and share data with other apps
- Resource Manager - Provides access to non-code embedded resources such as string , color settings , and user interface layouts
Notifications Manager - Allows applications to display alerts and notifications to the user
View system - set of view used to create application user interface

Application Components
- is the essential building blocks of Android application
- Components are loosely coupled by the application manifest (AndroidManifest.xml) that describes each component and how they interact.
4 main components
- Activities - dictate the UI and handle the user interaction to smartphone’s screen
- Services - handle background processing associated with the application
- Broadcast Receivers - handle combination btw Android OS and Applications
- Content Provider - handle data and database management issues
Activities
- represent a single screen with UI.
- perform action on screen
- For example , email application might have one activity that shows list of new mails.
another activity , to compose email
another is for reading emails
** An activity is implemented as a subclass of Activity class **
public class MainActivity extends Activity {
}
Services
- is a component that runs in background to perform long-running operations
- For example , a service to play music in the background while user if in another application
public class MyService extends Service {
}

Additional Components
- Fragments - represents a portion of user interface in an Activity.
- Views - UI elements that drawn on screen includes buttons , list , forms , etc...
- Layouts - View hierarchies that control screen format and appearance of the views.
- Intents - Message wiring components together.
- Resources - External elements , ex. string
- Manifest - Configuration file for the app
Anatomy of Android Application
- src - contain .java source files for your project.
- gen - contains the .R file , a compiler-generated file references all resources found in project. SHOULD NOT BE MODIFIED.
- bin - contain android package .apk built by the ADT during build process
- res/drawable-hdpi - for drawable object designed for high-density screen.
- res/layout - define app’s user interface.
- res/value - contain a collection of resources , ex. strings and color definitions.
- AndroidManifest.xml - describe fundamental characteristic of app
MainActivity

- gets converted to a Dalvik executable and runs the app.
Manifest File
you must declare all components in manifest.xml
- the file work as an interface between Android OS and your application
- if component not declared in this file , it will not be considered by the OS
action - action for intent filter is named android.intent.action.MAIN
indicate that this activity serves as entry point of application.
category for intent filter named android.intent.category.LAUNCHER to indicate that application can be launched

strings.xml

the R file

anim/ - XML file that define proper animations.
color/ - XML files that define a state list of color
drawable - image file or XML file that compile into bitmaps , state lists , shapes , animation drawable.
layout/ - XML files define interface layout
menu/ - XML files define application menu , options menu , context menu
raw/ - Arbitrary files to save in their raw form. call Resource.openRawResource() with resource ID to open raw file

xml/ - Arbitrary XML file that can be read at runtime by calling Resource.getXML()
Accessing Resources
- access defined resources in the code or in layout XML files.
Accessing Resources in Code
- when your Android app is compiled , R class get generated , contain resource IDs for all resources available in your res directory.
- use R class to access that resource using sub-directory and resource name or directly resource ID
Example
- to access res/drawable/myimage.png and set ImageView
ImageView imageView = (ImageView) findViewById(R.id.myimageview);
imageView.setImageResource(R.drawable.myimage);
- first line , use of R.id.myimageview to get ImageView defined with id myimageview in Layout file.
- second line , use of R.drawable.myimage to get an image with name my image available in drawable set-directory under /res.
Android - Activities
- an Activity represents a single screen with user interface. like window or frame of java.
- subclass of ContextThemeWrapper class.
android system initiates its program with in an Activity starting with a call on onCreate() method.
onCreate()
first callback and called when activity is first created
onStart()
callback method called when activity becomes visible to user
onResume()
called when user interact with applications
onPause()
does not receive user input and can not execute any code. called when current activity is being paused and previous activity is being resumed
onStop()
callback called when activity is no longer visible
onDestroy()
called before the activity is destroyed by the system.
onRestart()
called when activity restarts after stopping it.
- main activities must declared in AndroidManifest.xml with an <intent-filter> that includes MAIN and LAUNCHER.
Service
component that runs in the background.
- no need to interact to user
Service take two states
Started - when component ,ex activity, start calling startService().
- once start , a service can run indefinitely , even if the component started its destroy method.
Bound - when application component bind to it by calling bindService().
- offers a client-server interface allows components to interact with service , send request , get result, and IPC
(interprocess communication) - do so across process.
